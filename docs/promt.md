# Промт:

## Необходимо разработать схему базы данных PostgreSQL (для Supabase) для приложения "Дневник дегустаций вина".

### Требования к функционалу:

- Пользователи: Хранить ID Telegram и ID в системе. У каждого пользователя есть профиль (ник, аватар, соцсети, приватность).
- Справочники: Справочники стран, регионов, производителей, типов вин, сортов винограда, цветов, ароматов и универсальный справочник уровней (кислотность, танины и т.д.).
- Вина: Таблица вин с привязкой к справочникам. Возможность указать "свое значение" (custom), если в справочнике нет нужного.
- Дегустации: Таблица записей дегустаций. Привязка к вину и пользователю. Хранит дату, оценку (0-10), цену, фото, органолептику (цвет, аромат, вкус).
- Аналитика и статистика:
  - Таблица wine_rating_stats для хранения предрасчитанной статистики по каждому вину (средняя оценка, мин/макс, дисперсия, распределение оценок). Обновляется триггером.
  - Таблица user_wine_stats для хранения статистики "сколько раз пользователь пробовал это вино". Обновляется триггером.
  - Фото: Хранить ссылки на фото (Supabase Storage). Одна фотография может быть главной.
Технические требования:

-Использовать uuid как первичные ключи.
-Все поля, где возможен выбор из справочника ИЛИ ручной ввод, должны иметь пару полей: field_id (ссылка) и field_custom (текст).
-Написать триггеры и функции PL/pgSQL для автоматического пересчета статистики при добавлении/изменении/удалении дегустаций.
-Добавить подробные комментарии к каждой таблице и полю.
-Обеспечить целостность данных через Foreign Keys и Constraints.
